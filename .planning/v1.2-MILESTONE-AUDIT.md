## Integration Gaps (Critical)

### 1. AOFService does not forward executor to ProtocolRouter

**Severity:** Critical
**File:** `src/service/aof-service.ts` lines 108-114
**Affects:** EXEC-03 (completion-triggered advancement)

`AOFService` constructs `ProtocolRouter` without passing `executor` or `spawnTimeoutMs`. The `ProtocolRouter.handleSessionEnd()` dispatch path is dead code — `this.executor` is always `undefined` in production.

**Impact:** Hop advancement only occurs on the next scheduler poll cycle, not immediately on session completion. The fallback works but adds latency equal to one poll interval per hop transition.

**Fix:**
```typescript
// src/service/aof-service.ts — add executor and spawnTimeoutMs
this.protocolRouter = deps.protocolRouter ?? new ProtocolRouter({
  store: storeWithHooks,
  logger: this.logger,
  notifier: this.notifier,
  projectStoreResolver,
  cascadeBlocks: config.cascadeBlocks,
  executor: deps.executor,                // ADD
  spawnTimeoutMs: config.spawnTimeoutMs,   // ADD
});
```

### 2. migrateGateToDAG called without workflowConfig

**Severity:** Critical
**File:** `src/store/task-store.ts` lines 226 and 299
**Affects:** SAFE-05 (gate-to-DAG lazy migration)

`migrateGateToDAG(task)` is called without the `workflowConfig` second argument. The function guards on missing config and returns the task unchanged with a `console.warn`. Migration is a no-op for all real gate tasks.

**Impact:** Gate-format tasks are never migrated to DAG format. `docs/guide/migration.md` lines 222-224 incorrectly claims automatic migration works.

**Fix:** Load the project manifest's workflow config and pass as second argument in both `get()` and `list()` hooks.
