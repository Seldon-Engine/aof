## Integration Assessment

**Phase 10 internal integration:** All 10 DAG schema exports correctly wired (workflow-dag.ts → task.ts → index.ts → top-level barrel). Zero orphaned exports.

**Cross-phase integration gaps (expected — future phases):**
- `validateDAG()` has no call site outside tests — Phase 12 will wire it into task creation
- `initializeWorkflowState()` has no call site outside tests — Phase 12 will wire it into task creation
- `ITaskStore.create()` has no `workflow` parameter — Phase 12 will extend the interface
- Scheduler has zero references to `HopStatus` or `WorkflowState` — Phase 11/12 will add hop-level dispatch

**E2E flows:**
- Create DAG task: Breaks at store API (no workflow parameter) — Phase 12 scope
- Advance hop lifecycle: Breaks at scheduler (no hop concept) — Phase 11/12 scope

**Notable structural finding:** Timeout regex divergence between gate workflows (`/^\d+[mh]$/` — minutes/hours only) and DAG hops (`/^\d+[mhd]$/` — adds days). Intentional per SUMMARY — Phase 15 migration should account for this.
