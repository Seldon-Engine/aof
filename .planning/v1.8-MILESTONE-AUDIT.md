## Integration Gaps

### 1. deliverAllGranularityCallbacks Not Wired (GRAN-02)

**From:** Phase 31 `callback-delivery.ts:189` (exports `deliverAllGranularityCallbacks`)
**To:** Phase 30-02 `assign-executor.ts` `onRunComplete` (lines 205-224, 309-327)

The function exists, is exported, and has full test coverage. But no production code ever calls it. `onRunComplete` only calls `deliverCallbacks`, which filters for `completion` granularity (line 69). All-granularity subscribers are only served by the `retryPendingDeliveries` scan on terminal tasks, effectively degrading to completion-only with a poll delay.

**Fix:** Add `deliverAllGranularityCallbacks` call in both `onRunComplete` branches in `assign-executor.ts`.

### 2. callbackDepth Not Propagated Through MCP Boundary (SAFE-01)

**From:** Phase 31 sets `metadata.callbackDepth` on spawned session `TaskContext`
**To:** Phase 29 `handleAofDispatch` in `tools.ts` (line 115)

When a callback session dispatches a new task via `aof_dispatch`, the session's `metadata.callbackDepth` is never read or written to the new task's `frontmatter.callbackDepth`. The depth guard at `callback-delivery.ts:59` reads `frontmatter.callbackDepth` which will always be 0/undefined, making the infinite loop prevention ineffective.

**Fix:** `handleAofDispatch` needs to read `callbackDepth` from the session context and set it on new task frontmatter.
