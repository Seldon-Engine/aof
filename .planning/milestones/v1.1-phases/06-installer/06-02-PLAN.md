Create `scripts/install.sh` as a POSIX-compatible (`#!/bin/sh`) shell script for `curl -fsSL <url>/install.sh | sh` usage.

**Structure:** Follow the function-wrapped pattern (all logic in `main()`, called at end) to prevent partial download execution. Use `set -eu` for safety.

**Arguments:**
- `--prefix <path>` -- override install directory (default: `$HOME/.aof`)
- `--version <ver>` -- specific version to install (default: `latest`)
- `--openclaw-path <path>` -- explicit OpenClaw config path (skip heuristic)
- `--channel <ch>` -- release channel: stable/beta (default: stable)

**Implementation -- main() flow:**

1. **parse_args** -- loop over `$@`, set `INSTALL_DIR`, `VERSION`, `OPENCLAW_PATH`, `CHANNEL`

2. **check_prerequisites** -- in this order:
   - `node` exists and version >= 22 (parse `node --version`, compare major). On failure: print "Node.js >= 22 required" with install link, exit 1.
   - `tar` exists. On failure: print how to install, exit 1.
   - `curl` or `wget` exists (set `DOWNLOAD_CMD` for later). On failure: print need one of them, exit 1.
   - `git` exists -- soft check, warn if missing but continue (git used for some features but not install).
   - Write permissions on `$INSTALL_DIR` parent directory. On failure: print permission error, exit 1.
   - Disk space -- soft check via `df`, warn if < 500MB.

3. **detect_existing_install** -- check for:
   - `$INSTALL_DIR/.version` file (modern install) -- read version, set `EXISTING_VERSION`
   - `$HOME/.openclaw/aof/package.json` (legacy install at old DEFAULT_DATA_DIR) -- set `LEGACY_INSTALL=true`
   - `$HOME/.openclaw/extensions/aof/` (legacy extensions path) -- set `LEGACY_INSTALL=true`
   - If any detected: set `IS_UPGRADE=true`, print "Existing installation detected (vX.Y.Z)"

4. **determine_version** -- if `$VERSION` is `latest`:
   - Use curl/wget to query `https://api.github.com/repos/demerzel-ops/aof/releases/latest`
   - Parse `tag_name` field using sed/grep (no jq dependency)
   - Strip `v` prefix
   - Set `VERSION` and `TAG`

5. **download_tarball** -- download to temp file:
   - URL: `https://github.com/demerzel-ops/aof/releases/download/v${VERSION}/aof-v${VERSION}.tar.gz`
   - Use `curl -fsSL -o` or `wget -q -O` depending on `DOWNLOAD_CMD`
   - Register temp file in `CLEANUP_PATHS` for trap cleanup

6. **extract_and_install**:
   - Create `$INSTALL_DIR` if fresh install
   - If upgrade: back up data directories (tasks/, events/, memory/, state/, data/, .aof/, memory.db, memory-hnsw.dat) to `$INSTALL_DIR/.aof-backup/backup-$(date +%s)/`
   - Extract tarball: `tar -xzf "$TARBALL" -C "$INSTALL_DIR"`
   - Run `npm ci --production` in `$INSTALL_DIR` (or `npm install --production` if no lockfile)
   - If upgrade: restore data directories from backup

7. **run_node_setup** -- delegate complex logic to Node.js:
   ```sh
   node "$INSTALL_DIR/dist/cli/index.js" setup --auto \
     --data-dir "$INSTALL_DIR" \
     ${IS_UPGRADE:+--upgrade} \
     ${LEGACY_INSTALL:+--legacy} \
     ${OPENCLAW_PATH:+--openclaw-path "$OPENCLAW_PATH"} \
     2>&1
   ```

8. **write_version_file** -- write `$VERSION` to `$INSTALL_DIR/.version`

9. **print_summary** -- output:
   ```
   AOF v{VERSION} installed successfully!

   Location: ~/.aof
   {if upgrade: Upgraded from v{OLD} to v{NEW}}
   {if openclaw wired: OpenClaw plugin: configured}
   {if openclaw not found: OpenClaw: not detected (install OpenClaw to use AOF as a platform plugin)}

   Next steps:
     1. Review your org chart: ~/.aof/org/org-chart.yaml
     2. Start the daemon:      aof daemon install && aof daemon start
     3. Create your first task: aof task create "My first task"
   ```

**Trap-based cleanup:**
- Register `cleanup()` on `EXIT` trap
- Track `CLEANUP_PATHS` (temp tarball, any partially-created directories on fresh install)
- On fresh install failure: remove `$INSTALL_DIR` entirely (user decision: leave machine clean)
- On upgrade failure: restore from backup, print error

**Output helpers:**
- `say()` -- print with green checkmark: `printf "  \033[32m✓\033[0m %s\n" "$1"`
- `warn()` -- print with yellow warning: `printf "  \033[33m!\033[0m %s\n" "$1"`
- `err()` -- print with red cross: `printf "  \033[31m✗\033[0m %s\n" "$1"` to stderr

**Critical:** The script must be fully automatic -- zero interactive prompts (user decision). Every decision point has a default. If OpenClaw is not found, install AOF anyway, skip wiring, print warning.

**src/cli/commands/setup.ts** -- Create a new CLI command `aof setup` that orchestrates the post-extraction installation steps. This is called by install.sh after tarball extraction and npm ci.

The command accepts these flags:
- `--auto` -- fully automatic mode, no prompts (required for pipe-friendly install)
- `--data-dir <path>` -- AOF root directory (default: `~/.aof`)
- `--upgrade` -- existing install detected, run upgrade flow
- `--legacy` -- legacy install detected at ~/.openclaw/aof/
- `--openclaw-path <path>` -- explicit OpenClaw config path
- `--template <minimal|full>` -- org chart template (default: `minimal`)

**Implementation flow:**

```typescript
export async function runSetup(opts: SetupOptions): Promise<void> {
  const { dataDir, auto, upgrade, legacy, openclawPath, template } = opts;

  // 1. If legacy install detected, migrate data from ~/.openclaw/aof/ to dataDir
  if (legacy) {
    await migrateLegacyData(dataDir);
  }

  // 2. If upgrade: run migrations
  if (upgrade || legacy) {
    const currentVersion = readVersionFile(dataDir); // from .version file or "0.0.0" for legacy
    const targetVersion = readPackageVersion(dataDir); // from package.json in install dir
    const result = await runMigrations({
      aofRoot: dataDir,
      migrations: getAllMigrations(), // import registered migrations
      targetVersion,
    });
    say(`Migrations: ${result.applied.length} applied`);
  }

  // 3. If fresh install: run wizard for workspace scaffolding
  if (!upgrade && !legacy) {
    const result = await runWizard({
      installDir: dataDir,
      template: template || "minimal",
      interactive: !auto,
      skipOpenClaw: true, // We handle OpenClaw wiring separately below
      healthCheck: true,
      force: false,
    });
    say(`Workspace scaffolded: ${result.created.length} items created`);
  }

  // 4. OpenClaw plugin wiring
  await wireOpenClawPlugin(dataDir, openclawPath);
}
```

The `wireOpenClawPlugin()` helper:
1. Import `detectOpenClaw` from `openclaw-cli.ts` (NOT from wizard.ts -- use the CLI-based one)
2. If `openclawPath` provided: validate it exists, use it
3. Otherwise: call `detectOpenClaw()` for heuristic detection
4. If OpenClaw not found: `warn("OpenClaw not detected. Skipping plugin wiring.")` and return (soft requirement per user decision)
5. If OpenClaw found:
   - Call `isAofPluginRegistered()` -- if already registered, update; if not, register new
   - Call `registerAofPlugin()` with path to openclaw.plugin.json in dataDir
   - Call `detectMemoryPlugin()` to find existing memory plugin
   - Call `configureAofAsMemoryPlugin(memInfo.slotHolder)` to set AOF as memory plugin and disable existing one
   - Set `plugins.load.paths` to include dataDir: check existing paths via `openclawConfigGet("plugins.load.paths")`, add dataDir if not present, remove old AOF paths
   - Set `plugins.entries.aof.config.dataDir` to dataDir via `openclawConfigSet`
   - Run health check: `openclawConfigGet("plugins.entries.aof")` to verify the entry exists
   - If health check fails: roll back by unsetting the config entries, warn user, but keep files installed

The `migrateLegacyData()` helper:
1. Check `$HOME/.openclaw/aof/` for data directories (tasks/, events/, memory/, data/, state/)
2. Copy them to `dataDir` (do NOT move -- keep originals as implicit backup)
3. Print what was migrated
4. Write `.version` file with `0.0.0` to mark as migrated legacy

The `getAllMigrations()` function:
- Returns an empty array for now (no migrations registered yet). The framework is in place; actual migrations will be added when schema changes happen.
