  <action>
    Create `scripts/check-docs.mjs` implementing four checks. The script runs all checks, collects failures, and exits non-zero if any fail. Use ES module syntax.

    **Check 1: Stale generated docs**
    - Run the doc generator to a temporary file (use `os.tmpdir()`)
    - Compare temp output against committed `docs/guide/cli-reference.md`
    - If content differs: FAIL with message "Generated CLI docs are stale. Run: npm run docs:generate"
    - If `dist/cli/program.js` doesn't exist: FAIL with message "Build required. Run: npm run build"
    - Clean up temp file after comparison
    - IMPORTANT: Do NOT modify the committed file — only compare

    **Check 2: New commands without docs**
    - Import program from `../dist/cli/program.js`
    - Walk the command tree to get all command names (same recursion as generator)
    - Read `docs/guide/cli-reference.md` and extract all `### \`aof ...\`` headings
    - If any command exists in the tree but not in the doc headings: FAIL with list of undocumented commands
    - Skip the 'help' command

    **Check 3: Broken internal links**
    - Glob all .md files in docs/ and the root README.md and CONTRIBUTING.md
    - For each file, extract `[text](target)` links using regex
    - Skip external links (http/https), anchor-only links (#), and mailto: links
    - Resolve each relative link against the file's directory
    - Strip fragment (#...) before resolving
    - If resolved path doesn't exist: record as broken
    - Report all broken links and FAIL if any found

    **Check 4: README freshness**
    - Read `package.json` for: `version`, `repository.url`, `engines.node`
    - Read `README.md`
    - Check that:
      a. Repository URL in README matches package.json (extract domain/path from both, compare)
      b. Node.js version prerequisite in README is consistent with engines.node (README should say ">= 22" or "22+", not "20+")
    - FAIL if any mismatch found, with specific message about what's wrong

    **Script structure:**
    ```javascript
    const checks = [
      { name: 'Stale generated docs', fn: checkStaleDocs },
      { name: 'New commands without docs', fn: checkUndocumentedCommands },
      { name: 'Broken internal links', fn: checkBrokenLinks },
      { name: 'README freshness', fn: checkReadmeFreshness },
    ];

    let failed = false;
    for (const check of checks) {
      try {
        const issues = await check.fn();
        if (issues.length > 0) {
          console.error(`FAIL: ${check.name}`);
          for (const issue of issues) console.error(`  - ${issue}`);
          failed = true;
        } else {
          console.log(`OK: ${check.name}`);
        }
      } catch (err) {
        console.error(`ERROR: ${check.name}: ${err.message}`);
        failed = true;
      }
    }
    if (failed) process.exit(1);
    ```

    **Update package.json** — add pre-commit to simple-git-hooks:
    ```json
    "simple-git-hooks": {
      "commit-msg": "npx --no -- commitlint --edit $1",
      "pre-commit": "node scripts/check-docs.mjs"
    }
    ```

    **Run `npx simple-git-hooks`** after updating package.json to install the new hook.

    **Performance target:** The entire hook should run under 3 seconds (per research Pitfall 2). The generator imports a built module and does string comparison — no rebuild needed.
