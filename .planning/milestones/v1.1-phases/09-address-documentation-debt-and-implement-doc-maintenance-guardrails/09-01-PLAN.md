The refactor splits this into:
1. src/cli/program.ts — everything EXCEPT the parseAsync call. Exports `program`.
2. src/cli/index.ts — imports program, calls program.parseAsync()

Commander.js 14.0.3 introspection API:
- program.commands: Command[] — array of registered subcommands
- cmd.name(): string — command name
- cmd.description(): string — command description
- cmd.options: Option[] — registered options (each has .flags, .description, .defaultValue)
- cmd.registeredArguments: Argument[] — registered args (each has .name(), .required, .description)
- cmd.commands: Command[] — nested subcommands (recursive)
</interfaces>

    ```typescript
      process.exitCode = 1;
    });
    ```

    The shebang line `#!/usr/bin/env node` MUST be on `index.ts` (not program.ts) since index.ts is the bin entrypoint.

    Create `scripts/generate-cli-docs.mjs` that:

    1. Imports the program from `../dist/cli/program.js` (requires prior build)
    2. Checks that `dist/cli/program.js` exists — if not, prints "Run `npm run build` first" and exits 1
    3. Recursively walks the command tree using this algorithm:
       - For each command (skip 'help'):
         - Emit `### \`aof {fullName}\`` heading (e.g., `### \`aof daemon start\``)
         - Emit description
         - If registeredArguments.length > 0: emit arguments table (Argument | Required | Description)
         - If options.length > 0 (filter hidden): emit options table (Flag | Description | Default)
         - Recurse into cmd.commands
    4. Wraps output in a document with:
       - Comment: `<!-- AUTO-GENERATED — do not edit manually. Run: npm run docs:generate -->`
       - Title: `# CLI Reference`
       - Brief intro paragraph
       - Table of contents (list of command links)
       - All command sections
    5. Writes output to `docs/guide/cli-reference.md`
    6. Creates `docs/guide/` directory if it doesn't exist (using `mkdirSync` with recursive)
    7. Prints the path of the generated file to stdout

    Add to package.json scripts:
    ```json
    "docs:generate": "node scripts/generate-cli-docs.mjs"
    ```

    Run the generator to produce the initial `docs/guide/cli-reference.md`.

    The script must use ES module syntax (import, not require) since the project is type: "module".
  </action>
