    - WorkflowDefinition: parses object with name + hops array (min 1 hop); rejects empty hops array
    - HopStatus: enum with exactly ["pending", "ready", "dispatched", "complete", "failed", "skipped"]
    - HopState: parses with status + optional startedAt, completedAt, agent, correlationId, result
    - WorkflowStatus: enum with exactly ["pending", "running", "complete", "failed"]
    - WorkflowState: parses with status + hops map + optional startedAt, completedAt
    - TaskWorkflow: parses with definition + state nested structure
    - validateDAG: returns [] for valid linear DAG (A -> B -> C)
    - validateDAG: returns [] for valid diamond DAG (A -> B, A -> C, B -> D, C -> D)
    - validateDAG: returns [] for valid parallel fan-out (A -> B, A -> C, A -> D)
    - validateDAG: returns error for cycle (A -> B -> C -> A)
    - validateDAG: returns error for self-referencing hop (A dependsOn A)
    - validateDAG: returns error for duplicate hop IDs
    - validateDAG: returns error for dangling dependsOn reference (references non-existent hop)
    - validateDAG: returns error for unreachable hops (hop with dependsOn pointing to valid hop but itself not reachable from any root)
    - validateDAG: returns error for no root hops (all hops have dependsOn)
    - validateDAG: validates timeout format (valid: "1h", "30m", "2h"; invalid: "abc", "1x")
    - validateDAG: returns error for empty escalateTo string

    - initializeWorkflowState: workflow-level status is "pending"
    - initializeWorkflowState: all hops from definition appear in state map

    1. **ConditionExpr** -- Recursive discriminated union via `z.lazy()`. Use `z.discriminatedUnion("op", [...])` with operators: eq, neq (field+value), gt, gte, lt, lte (field+number value), in (field+array value), has_tag (string value), hop_status (hop+status strings), and/or (conditions array, recursive), not (single condition, recursive), true, false (no args). Define a `ConditionExprType` interface for the recursive type annotation.

    2. **Hop** -- `z.object()` with fields per CONTEXT.md locked decisions:
       - `id: z.string().min(1)` -- unique within workflow
       - `role: z.string().min(1)` -- target role from org chart
       - `dependsOn: z.array(z.string()).default([])` -- predecessor hop IDs
       - `joinType: z.enum(["all", "any"]).default("all")` -- join semantics
       - `autoAdvance: z.boolean().default(true)` -- auto vs review
       - `condition: ConditionExpr.optional()` -- JSON DSL condition
       - `description: z.string().optional()` -- human-readable
       - `canReject: z.boolean().default(false)` -- Phase 13 placeholder
       - `rejectionStrategy: z.enum(["origin", "predecessors"]).optional()` -- Phase 13 placeholder
       - `timeout: z.string().optional()` -- Phase 13 placeholder
       - `escalateTo: z.string().optional()` -- Phase 13 placeholder

    3. **WorkflowDefinition** -- `z.object({ name: z.string().min(1), hops: z.array(Hop).min(1) })`

    4. **HopStatus** -- `z.enum(["pending", "ready", "dispatched", "complete", "failed", "skipped"])`

    5. **HopState** -- `z.object()` with status, optional startedAt/completedAt (datetime), optional agent/correlationId (string), optional result (record of string to unknown)

    6. **WorkflowStatus** -- `z.enum(["pending", "running", "complete", "failed"])`

    7. **WorkflowState** -- `z.object()` with status (WorkflowStatus), hops (record of string to HopState), optional startedAt/completedAt (datetime)

    8. **TaskWorkflow** -- `z.object({ definition: WorkflowDefinition, state: WorkflowState })`

    9. **validateDAG(definition)** -- Standalone function returning `string[]` (same pattern as `validateWorkflow()` in workflow.ts). Checks:

       - All dependsOn references exist
       - At least one root hop (empty dependsOn)
       - No cycles (Kahn's algorithm -- ~25 lines, see RESEARCH.md Example 4)
       - All hops reachable from root hops (BFS from roots, check all visited)
       - Timeout format validation (`/^\d+[mhd]$/` -- minutes, hours, days)
       - escalateTo non-empty if present
       Do NOT put graph validation in Zod `.superRefine()` -- keep it in standalone function per RESEARCH.md Pitfall 3 (avoids slow parse on every task load).

    10. **initializeWorkflowState(definition)** -- Pure helper returning `WorkflowState`. Root hops (dependsOn empty) get status "ready", others get "pending". Workflow-level status is "pending". No timestamps.
