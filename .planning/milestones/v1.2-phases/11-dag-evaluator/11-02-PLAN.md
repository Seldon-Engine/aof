<feature>
  <name>DAG Evaluator</name>
  <files>src/dispatch/dag-evaluator.ts, src/dispatch/__tests__/dag-evaluator.test.ts, src/dispatch/index.ts</files>
  <behavior>
    --- Input/Output Contract ---
    - evaluateDAG(input: DAGEvaluationInput) -> DAGEvaluationResult
    - Input: { definition: WorkflowDefinition, state: WorkflowState, event: HopEvent, context: EvalContext }
    - HopEvent: { hopId: string, outcome: "complete" | "failed" | "skipped", result?: Record<string, unknown> }
    - EvalContext: { hopResults: Record<string, Record<string, unknown>>, task: { status, tags, priority, routing } }
    - Result: { state: WorkflowState (new), changes: HopTransition[], readyHops: string[], dagStatus?: WorkflowStatus, taskStatus?: string }
    - HopTransition: { hopId: string, from: HopStatus, to: HopStatus, reason?: string }
    - Input state is NEVER mutated (use structuredClone)

    --- Primary Event Application ---
    - Apply hop event: update hopId's status to complete/failed/skipped with completedAt timestamp and result data
    - Record the transition in changes array

    --- Skip Cascade (EXEC-05) ---
    - After a hop fails or is skipped, cascade to downstream hops
    - A downstream hop auto-skips ONLY if ALL its predecessors are in terminal non-success state (skipped or failed)
    - If ANY predecessor completed, the hop can still proceed (no skip)
    - Cascade is fully recursive in a single call: A skips -> B skips -> C skips, all in one changes array
    - Only skip hops that are currently "pending" (don't re-skip already terminal hops)

    --- Condition Evaluation ---
    - After primary event + skip cascade, evaluate conditions on newly eligible hops
    - A hop is eligible for condition evaluation when: it is pending AND all its predecessors are terminal
    - If condition evaluates to false, mark hop as "skipped" with reason "condition" and cascade skips downstream
    - If condition evaluates to true, leave hop for readiness determination

    --- Readiness Determination (EXEC-07) ---
    - AND-join (joinType: "all"): hop becomes ready when ALL predecessors are complete or skipped (none pending/dispatched)
    - AND-join edge case: if ALL predecessors are terminal non-success (all skipped/failed), the hop should have been cascade-skipped, not marked ready
    - OR-join (joinType: "any"): hop becomes ready when ANY predecessor has status "complete" (only complete triggers, not skip/fail)
    - OR-join edge case: if ALL predecessors are terminal and none completed, OR-join should cascade-skip
    - Root hops (empty dependsOn) are NOT handled here (handled by initializeWorkflowState at creation time)
    - Mark newly ready hops with status "ready" in the state and record transitions

    --- DAG Completion ---
    - Check after all cascades and readiness updates are done
    - DAG is "complete" when ALL hops are terminal (complete or skipped, no failed) -- no pending, ready, or dispatched
    - DAG is "failed" when ALL hops are terminal and at least one is "failed"
    - If any hops are still pending/ready/dispatched, DAG status stays unchanged (still running)
    - Set workflow completedAt timestamp on completion/failure
    - Suggest taskStatus: "done" for complete, "failed" for failed, undefined otherwise

    --- Evaluator Pipeline Ordering ---
    1. Apply primary hop event
    2. If hop failed/skipped, cascade skips downstream
    3. Evaluate conditions on newly eligible hops, cascade any new skips
    4. Determine newly ready hops
    5. Check DAG completion
    6. Return new state + change summary

    --- Barrel Export ---
    - src/dispatch/index.ts exports evaluateDAG, DAGEvaluationInput, DAGEvaluationResult, HopEvent, HopTransition, EvalContext
    - src/dispatch/index.ts exports evaluateCondition, getField, buildConditionContext, ConditionContext from dag-condition-evaluator
  </behavior>
  <implementation>
    Implement `evaluateDAG()` as the single exported pure function in `src/dispatch/dag-evaluator.ts`, mirroring the gate-evaluator.ts pattern. Internal helpers:

    1. `buildDownstreamIndex(definition)` -- reverse adjacency map: hopId -> downstream hopIds (from dependsOn edges)
    2. `cascadeSkips(definition, state, changes, downstreamIndex, startHopId, timestamp)` -- recursive downstream skip propagation
    3. `evaluateNewlyEligibleConditions(definition, state, changes, downstreamIndex, conditionContext, timestamp)` -- find hops where all predecessors are terminal, evaluate conditions, cascade skips for condition-skipped hops
    4. `determineReadyHops(definition, state)` -- scan pending hops with satisfied join conditions
