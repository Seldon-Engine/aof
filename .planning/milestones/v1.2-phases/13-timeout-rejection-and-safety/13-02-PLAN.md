  <files>src/dispatch/escalation.ts, src/dispatch/__tests__/dag-timeout.test.ts, src/dispatch/scheduler.ts</files>
  <behavior>
    **checkHopTimeouts() — scanning for timed-out hops:**
    - Given an in-progress task with a DAG workflow and a dispatched hop where elapsed time > hop.timeout → returns an escalation action
    - Given a dispatched hop with no timeout configured → skips it (no action)
    - Given a dispatched hop that has NOT exceeded its timeout → skips it (no action)
    - Given a task with no workflow (gate-based) → skips it (no action)
    - Given a hop with status other than "dispatched" → skips it (no action)

    **escalateHopTimeout() — escalation behavior:**
    - Given hop has escalateTo role → force-completes timed-out session, sets hop agent to escalateTo, marks escalated=true on HopState, resets startedAt for new timer, clears correlationId, persists state, logs "dag.hop_timeout_escalation" event, returns alert action with escalation info
    - Given hop has NO escalateTo role → logs "dag.hop_timeout" event, returns alert action with warning, does NOT modify hop state
    - Given hop already has escalated=true (one-shot rule) → logs "dag.hop_timeout" event, returns alert action, does NOT re-escalate

    **Force-complete behavior:**
    - Force-complete uses config.executor.forceCompleteSession(hopState.correlationId)
    - Force-complete happens BEFORE state persistence and re-dispatch (per CONTEXT.md decision)
    - If no executor or no correlationId, skip force-complete but continue with state update

    **Poll cycle integration:**
    - checkHopTimeouts is called in poll() alongside checkGateTimeouts
    - Returns SchedulerAction[] that gets merged into poll result actions

    **Edge cases:**
    - Hop with startedAt missing (defensive) → skip (can't calculate elapsed)
    - Hop with invalid timeout format (parseDuration returns null) → skip with warning log
    - dryRun mode → returns alert actions but does NOT force-complete, modify state, or dispatch
  </behavior>
