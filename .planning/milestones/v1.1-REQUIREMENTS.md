# Requirements: AOF v1.1 Stabilization & Ship

**Defined:** 2026-02-26
**Core Value:** Tasks never get dropped — they survive gateway restarts, API failures, rate limits, and agent crashes, always resuming and completing end-to-end without human intervention.

## v1.1 Requirements

Requirements for v1.1 milestone. Each maps to roadmap phases.

### Memory Fix

- [x] **MEM-01**: HNSW index resizes automatically when capacity is reached (no insert crashes)
- [x] **MEM-02**: HNSW-SQLite parity check runs on startup, logs mismatches
- [x] **MEM-03**: HNSW index rebuilds from SQLite when index file is missing or corrupt
- [x] **MEM-04**: Memory search returns correct results after index resize and after save/load cycle
- [x] **MEM-05**: `aof memory health` shows index count, SQLite count, fragmentation %, last rebuild time
- [x] **MEM-06**: `aof memory rebuild` forces full HNSW rebuild from SQLite with progress output

### CI Pipeline

- [x] **CI-01**: GitHub Actions workflow runs typecheck, build, and tests on every PR to main
- [x] **CI-02**: Pre-existing test failures (~12) are fixed so CI passes green
- [x] **CI-03**: Manual release workflow runs release-it, creates GitHub Release with changelog
- [x] **CI-04**: CHANGELOG.md is persisted in repo and updated on each release

### Installer

- [x] **INST-01**: `curl | sh` installer detects OS, architecture, Node >= 22, existing OpenClaw
- [x] **INST-02**: Installer downloads release tarball from GitHub and extracts correctly (`extractTarball()` fixed)
- [x] **INST-03**: Installer runs wizard (directory scaffolding, org chart template, health check)
- [x] **INST-04**: Installer auto-detects OpenClaw gateway and wires AOF as plugin
- [x] **INST-05**: Running installer on existing install upgrades without losing tasks/events/memory data
- [x] **INST-06**: Channels.ts repo URL points to real GitHub repository

### Projects (existing primitive — verification & gap-filling)

- [x] **PROJ-01**: ToolContext includes `projectId` field, tools scope operations to active project
- [x] **PROJ-02**: Task dispatch passes project ID to task store, tasks land in correct project directory
- [x] **PROJ-03**: Dispatcher filters eligible agents by project participants list
- [x] **PROJ-04**: Memory search respects project pool isolation (no cross-project results)
- [x] **PROJ-05**: `aof project create --template` scaffolds project directory with manifest, task dirs, and memory config
- [x] **PROJ-06**: Integration tests verify end-to-end project routing (create project, create task, dispatch, verify isolation)

### Documentation & Guardrails

- [x] **DOC-01**: `docs/guide/` contains end-user documentation; `docs/dev/` contains contributor/architecture documentation; existing flat docs files are relocated to correct audience directory
- [x] **DOC-02**: `docs/guide/getting-started.md` walks a new user from install to running their first task end-to-end
- [x] **DOC-03**: `docs/guide/configuration.md` covers org-chart.yaml schema, AOF config options, and OpenClaw plugin wiring
- [x] **DOC-04**: `docs/guide/cli-reference.md` is auto-generated from Commander.js command tree via `npm run docs:generate`
- [x] **DOC-05**: Root `README.md` is a concise landing page with correct repo URL, Node 22+ prerequisite, installer command, and links into `docs/`
- [x] **DOC-06**: `docs/dev/architecture.md` provides system overview for contributors
- [x] **DOC-07**: Pre-commit hook blocks commits when generated docs are stale, new commands lack doc entries, internal markdown links are broken, or README references are inconsistent with package.json
- [x] **DOC-08**: JSDoc on public API exports in `src/tools/` and `src/schemas/protocol.ts`

## Future Requirements (v1.2+)

### Polish

- **POL-02**: Mailbox view improvements

## Out of Scope

| Feature | Reason |
|---------|--------|
| OpenClaw gateway modification | AOF integrates with it, doesn't modify it |
| UI/dashboard | CLI and JSONL observability sufficient for v1 |
| Multi-host distribution | Single-machine deployment for now |
| Non-OpenClaw runtimes | AOF is an OpenClaw plugin specifically |
| npm publish | Distribution via GitHub Releases + installer, not npm registry |
| Multi-platform native binaries | Require Node.js >= 22 as prerequisite instead |
| OpenTelemetry integration | Deferred to v2 |
