- `daemon.ts` registers SIGTERM/SIGINT → `drainAndExit()`
- `daemon/index.ts` registers its own SIGTERM/SIGINT → `shutdown()`
- Both fire on same signal: concurrent drain, duplicate `process.exit(0)`
- Evidence: `MaxListenersExceededWarning: 11 SIGTERM listeners` in test suite

**2. Standalone daemon missing executor (GATE-01, GATE-02 integration)**
- `startAofDaemon()` constructs `AOFService` without passing `executor`
- `resolveAdapter()` only called in plugin mode (`registerAofPlugin`)
- Standalone daemon can poll but never dispatch — logs "no executor configured"
