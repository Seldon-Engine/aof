# Requirements: AOF Production Readiness

**Defined:** 2026-02-25
**Core Value:** Tasks never get dropped -- they survive gateway restarts, API failures, rate limits, and agent crashes, always resuming and completing end-to-end without human intervention.

## v1 Requirements

Requirements for production release. Each maps to roadmap phases.

### Foundation

- [x] **FOUND-01**: Scheduler poll loop has configurable timeout guard preventing infinite hangs
- [x] **FOUND-02**: Daemon performs graceful drain on shutdown, completing in-flight task transitions
- [x] **FOUND-03**: On startup, scheduler reconciles orphaned leases and reclaims abandoned tasks
- [x] **FOUND-04**: Failures classified as transient (retry with backoff) vs permanent (deadletter immediately)

### Daemon

- [x] **DAEM-01**: `aof daemon install` generates launchd plist on macOS
- [x] **DAEM-02**: `aof daemon install` generates systemd unit on Linux
- [x] **DAEM-03**: Health server binds before PID file is written (consistent startup state)
- [x] **DAEM-04**: Health endpoint returns task counts, uptime, version, and component status
- [x] **DAEM-05**: Watchdog detects daemon failure and performs actual restart (not stub)

### Gateway

- [x] **GATE-01**: `GatewayExecutor` dispatches tasks to agents via plugin-sdk adapter interface
- [x] **GATE-02**: Adapter interface abstracts platform-specific integration (OpenClaw first, portable later)
- [x] **GATE-03**: Dispatched sessions tracked from spawn to completion with correlation
- [x] **GATE-04**: Stuck agent sessions force-completed after configurable timeout
- [x] **GATE-05**: Integration test suite validates dispatch-to-completion E2E

### Self-Healing

- [ ] **HEAL-01**: Dead-lettered tasks automatically resurrected with exponential backoff
- [ ] **HEAL-02**: Stuck sessions detected via stale heartbeat and recovered
- [ ] **HEAL-03**: Per-provider circuit breaker prevents cascading failures (via cockatiel)

### Install

- [ ] **INST-01**: Competing `integration.ts` code path removed
- [ ] **INST-02**: Config validation errors are user-facing with fix suggestions (not raw Zod)
- [ ] **INST-03**: `aof init` works from any directory on any supported platform
- [ ] **INST-04**: Interactive install wizard via `@inquirer/prompts` guides full setup

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Observability

- **OBSV-01**: Correlation IDs thread through entire task lifecycle (create -> dispatch -> completion)
- **OBSV-02**: `aof daemon status` shows rich operational overview
- **OBSV-03**: OpenTelemetry integration for traces and metrics export

### Advanced Features

- **ADVF-01**: Agent-guided org chart setup (main agent interviews user, builds org-chart.yaml)
- **ADVF-02**: Model fallback chains defined in org chart
- **ADVF-03**: Dry-run mode for config changes (preview impact before applying)
- **ADVF-04**: Windows support for daemon lifecycle

### Multi-Platform

- **PLAT-01**: tinyclaw adapter for AOF dispatch
- **PLAT-02**: Claude Code adapter for AOF dispatch
- **PLAT-03**: zeroclaw adapter for AOF dispatch

## Out of Scope

| Feature | Reason |
|---------|--------|
| OpenClaw gateway modifications | AOF integrates via plugin-sdk, doesn't modify gateway |
| Web UI/dashboard | CLI + JSONL observability sufficient for v1 |
| Multi-host distribution | Single-machine deployment for v1 |
| LLM-based routing | Violates core principle: no LLMs in control plane |
| Database backend | Filesystem-based by design, atomic rename() is the API |
| Plugin ecosystem | AOF is the orchestrator, not a platform for plugins |
| Zod 4 / TypeScript 6 migration | Risk during stabilization phase, defer to post-v1 |
