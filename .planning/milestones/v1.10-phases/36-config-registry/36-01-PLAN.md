    - Test: invalid numeric env var (e.g., AOF_DEFAULT_LEASE_TTL_MS="abc") throws ConfigError listing the issue
    - Test: multiple invalid values throws ConfigError listing ALL issues, not just first
    - Test: ConfigError.issues contains Zod issue objects with path and message
    - Test: resetConfig() clears cache — next getConfig() re-reads env
    - Test: resetConfig({ core: { logLevel: "debug" } }) returns config with overridden logLevel but other defaults intact
    - Test: resetConfig with overrides produces a frozen result
    - Test: unknown AOF_* env var triggers console.warn with closest match suggestion
    - Test: AOF_ROOT is read as fallback when AOF_DATA_DIR is not set (backward compat)
    - Test: AOF_DATA_DIR takes precedence over AOF_ROOT when both set
    - Test: registry.ts imports nothing from dispatch/, service/, store/, protocol/ (CFG-04 — can verify via grep in verify step)
  </behavior>
  <action>
    Create `src/config/registry.ts` implementing:

    1. **AofConfigSchema** — Zod schema with nested domains per CONTEXT.md:
       - `core`: dataDir (string, default ~/.aof), logLevel (enum debug|info|warn|error|silent, default "info"), vaultRoot (string, optional)
       - `dispatch`: defaultLeaseTtlMs (coerce number, default 600000), spawnTimeoutMs (coerce number, default 120000), maxConcurrency (coerce int, default 3), maxDispatchesPerPoll (coerce int, default 10)
       - `daemon`: pollIntervalMs (coerce number, default 30000), socketPath (string, optional)
       - `openclaw`: gatewayUrl (string, default "http://localhost:3000"), gatewayToken (string, optional), stateDir (string, default "~/.openclaw"), configPath (string, optional)
       - `integrations`: openaiApiKey (string, optional)

    2. **readEnvInput()** — private function mapping process.env SCREAMING_SNAKE vars to nested camelCase structure. Read AOF_DATA_DIR first, fall back to AOF_ROOT for core.dataDir. Strip undefined values so Zod defaults apply. For vault vars: read AOF_VAULT_ROOT first, fall back to OPENCLAW_VAULT_ROOT.

    3. **ConfigError** — extends Error, takes ZodIssue[], formats as "Invalid AOF configuration:\n  - path: message" for each issue.

    4. **getConfig()** — lazy singleton. If cached, return cached. Otherwise: call readEnvInput(), safeParse with AofConfigSchema, throw ConfigError on failure, deepFreeze result, cache, call warnUnknownVars(), return.

    5. **resetConfig(overrides?)** — if no overrides, set cached = null. If overrides provided, create config from defaults deep-merged with overrides (do NOT read process.env), deepFreeze, cache. This is for test isolation.

    6. **warnUnknownVars()** — scan process.env for AOF_* keys not in KNOWN_AOF_VARS set. For each unknown, find closest known var by Levenshtein distance (simple inline implementation). Emit console.warn.

    7. **deepFreeze()** — recursive Object.freeze utility.

    Import only from: `zod`, `node:os`, `node:path`, and `./paths.js` (normalizePath only). This satisfies CFG-04.

    Export: getConfig, resetConfig, ConfigError, AofConfig (type), AofConfigSchema.

    Create `src/config/__tests__/registry.test.ts` with tests per behavior list above. Use vi.stubEnv() for env var manipulation. Call resetConfig() in afterEach to prevent test leakage.
