**Step 3: Wrap assign-executor operations in withLock**

In `src/dispatch/assign-executor.ts`:

1. Import `TaskLockManager` type.
2. In `executeAssignAction()` (the function that handles "assign" actions), wrap the entire per-task action body in `config.lockManager?.withLock(action.taskId, async () => { ... })` or execute directly if no lockManager:

```typescript
const executeBody = async () => {
  // existing: acquireLease, spawn, transition on failure — all current logic
};

if (config.lockManager) {
  await config.lockManager.withLock(action.taskId, executeBody);
} else {
  await executeBody();
}
```

The key call sites to wrap are:
- Line ~120: `acquireLease(store, action.taskId, ...)` + everything after (spawn, result handling)
- Line ~248: `store.transition(action.taskId, "blocked", ...)` (enforcement failure)
- Line ~439: `store.transition(action.taskId, "blocked", ...)` (spawn failure)

All three are within `executeAssignAction()`, so wrapping the whole function body is cleanest.

Also check `action-executor.ts` for the `expire_lease` case and other cases that call `store.transition` — wrap those too if they target a specific taskId. The expire_lease handler in action-executor.ts directly calls `store.transition` and `writeFileAtomic` for specific tasks — wrap each expire_lease iteration in `config.lockManager?.withLock(action.taskId, ...)`.
