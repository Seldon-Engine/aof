Add "all" granularity callback delivery with per-poll-cycle batching.

Purpose: Enable subscribers to track full task lifecycle (every state transition), not just terminal states. Transitions since last delivery are batched into a single callback per subscriber per poll cycle using lastDeliveredAt as a cursor into the event log.

Output: Extended schemas (lastDeliveredAt on subscription, new event types on event schema), deliverAllGranularityCallbacks function, and comprehensive tests.

<feature>
  <name>All-granularity callback delivery with batched transitions</name>
  <files>
    src/schemas/subscription.ts,
    src/schemas/event.ts,
    src/store/subscription-store.ts,
    src/dispatch/callback-delivery.ts,
    src/dispatch/__tests__/callback-delivery.test.ts
  </files>
  <behavior>
    - Schema: TaskSubscription gains `lastDeliveredAt: z.string().datetime().optional()` field
    - Schema: EventType enum gains `"subscription.depth_exceeded"` and `"subscription.recovery_attempted"` entries
    - Store: SubscriptionStore.update() Pick type expands to include `"lastDeliveredAt"`
    - deliverAllGranularityCallbacks(opts): For each active "all" subscription on a task, scans EventLogger for `task.transitioned` events after sub.lastDeliveredAt, batches them into ordered array of `{fromStatus, toStatus, timestamp}`, delivers single callback with transition list + current task state, updates lastDeliveredAt on success
    - When no new transitions found since lastDeliveredAt, skips delivery (no empty callbacks)
    - buildCallbackPrompt extended: accepts optional `transitions` array; when present, includes "## Transitions" section with ordered transition list instead of just final status
    - Delivery uses same best-effort pattern as existing deliverCallbacks (try/catch per subscriber, never propagates)
    - lastDeliveredAt updated to latest transition timestamp only after successful delivery

    Test cases (RED phase):
    - Test 1: "all" subscription with 3 transitions since lastDeliveredAt gets single callback with all 3 in order
    - Test 2: "all" subscription with no new transitions since lastDeliveredAt gets no callback
    - Test 3: lastDeliveredAt cursor advances to latest transition timestamp after successful delivery
    - Test 4: "all" subscription with lastDeliveredAt=undefined (first delivery) gets all transitions from task creation
    - Test 5: Failed delivery does NOT advance lastDeliveredAt (self-healing cursor)
    - Test 6: "all" subscription fires on terminal transitions too (superset of completion)
    - Test 7: buildCallbackPrompt includes transitions section when transitions array provided
    - Test 8: Schema validates lastDeliveredAt as optional datetime string
  </behavior>
  <implementation>
    1. Add lastDeliveredAt to TaskSubscription schema in subscription.ts
    2. Add subscription.depth_exceeded and subscription.recovery_attempted to EventType enum in event.ts (needed by plan 02, but schema changes grouped here for atomicity)
    3. Expand SubscriptionStore.update() Pick type to include lastDeliveredAt
    4. Extend buildCallbackPrompt to accept optional transitions array parameter; when provided, add "## Transitions" section listing each transition as "- {fromStatus} -> {toStatus} at {timestamp}"
    5. Implement deliverAllGranularityCallbacks in callback-delivery.ts:
       - Load task via store.get(taskId) -- note: does NOT check for terminal status (fires on every transition)
       - List active "all" granularity subscriptions
       - For each subscription:
         a. Compute lastDeliveredAtMs from sub.lastDeliveredAt (0 if undefined)
         b. Query logger for task.transitioned events filtered by taskId
         c. Filter events with timestamp > lastDeliveredAtMs
         d. Map to {fromStatus, toStatus, timestamp} ordered chronologically
         e. Skip if no new transitions
         f. Call deliverSingleCallback with transitions array (extend signature)
         g. On success, update subscription with lastDeliveredAt = latest transition timestamp
       - Wrap each subscriber delivery in try/catch (best-effort)
    6. Export deliverAllGranularityCallbacks from callback-delivery.ts
  </implementation>
</feature>
