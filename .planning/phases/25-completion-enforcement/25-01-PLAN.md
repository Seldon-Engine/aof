<objective>
Replace the auto-completion fallback with hard enforcement failure for both top-level tasks and DAG hops. When an agent exits without calling `aof_task_complete`, the task is blocked and tracked as a dispatch failure instead of being silently auto-completed.

Purpose: Stop trusting agent exit codes. Agents that hallucinate completion or fail silently are caught, retried, and eventually deadlettered — not passed through as successful.

Output: Modified `assign-executor.ts` with enforcement in `onRunComplete`, modified `dag-transition-handler.ts` with `onRunComplete` callback for DAG hops, new `completion.enforcement` event type, and tests covering both paths.

Note: ENFC-02 (configurable warn/block) is DROPPED per user decision — block-only, no configuration. ENFC-03 (no-op detection) is DEFERRED to Phase 26 per user decision.
</objective>
