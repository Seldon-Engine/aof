<objective>
Make the AOF scheduler lifecycle restart-safe: add a configurable timeout guard to the poll loop (FOUND-01), implement graceful drain on shutdown so in-flight transitions complete before exit (FOUND-02), and add startup reconciliation that reclaims orphaned tasks after a crash (FOUND-03).

Purpose: Without these, a hanging poll freezes the scheduler forever, a SIGTERM kills mid-transition leaving half-written state, and a SIGKILL leaves tasks stuck in in-progress with no recovery path.

Output: Modified `aof-service.ts` with timeout + drain + reconcile, updated `daemon.ts` with drain-aware signal handlers, and tests proving all three behaviors.
</objective>
