2. **Extend `classifySpawnError()` return type:**
   Change the return type from `"transient" | "permanent"` to `"transient" | "permanent" | "rate_limited"`:
   ```typescript
   export function classifySpawnError(error: string): "transient" | "permanent" | "rate_limited" {
     const lower = error.toLowerCase();
     for (const pattern of PERMANENT_ERROR_PATTERNS) {
       if (lower.includes(pattern)) return "permanent";
     }
     for (const pattern of RATE_LIMIT_PATTERNS) {
       if (lower.includes(pattern)) return "rate_limited";
     }
     return "transient";
   }
   ```
   NOTE: `rate_limited` is treated the same as `transient` for retry logic, but enables separate event logging and future circuit-breaker integration (Phase 4, HEAL-03).

3. **Update `shouldAllowSpawnFailedRequeue()` to handle `rate_limited`:**

   ```typescript
   // Permanent errors should never be retried
   if (errorClass === "permanent") { ... }
   // rate_limited treated same as transient for retry purposes (Phase 4 adds circuit breaker)
   ```

4. **Add jitter to `computeRetryBackoffMs()`:**
   Replace the current implementation (lines 161-165) with the jitter-enhanced version from RESEARCH.md Pattern 4:
   ```typescript
   export function computeRetryBackoffMs(retryCount: number, opts?: {
     baseMs?: number;
     ceilingMs?: number;
     jitterFactor?: number;
     jitterFn?: () => number;  // Defaults to Math.random, injectable for test determinism
   }): number {
     const baseMs = opts?.baseMs ?? 60_000;
     const ceilingMs = opts?.ceilingMs ?? 15 * 60_000;
     const jitterFactor = opts?.jitterFactor ?? 0.25;
     const random = opts?.jitterFn ?? Math.random;

     const delay = Math.min(baseMs * Math.pow(3, retryCount), ceilingMs);
     const jitter = delay * jitterFactor * (random() * 2 - 1);  // +/- jitterFactor
     return Math.max(0, Math.round(delay + jitter));
   }
   ```
   CRITICAL per RESEARCH.md Pitfall 4: The `jitterFn` parameter allows tests to inject a deterministic function (e.g., `() => 0.5`) instead of `Math.random()`. Without this, backoff tests will be flaky.
