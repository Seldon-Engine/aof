<objective>
Bug 2B — Routing validation at task creation. Today `aof_dispatch`
(the actual create+dispatch entry point — not the fictional
`aof_task_create` named in the post-mortem; RESEARCH.md confirmed)
accepts tasks with no `routing.agent`, `routing.role`, AND `routing.team`.
The scheduler rejects tags-only routing at dispatch time
(`src/dispatch/task-dispatcher.ts:191-250`), logging "task has tags-only
routing (not supported), needs explicit agent/role/team assignment" —
but the failure counter is NOT incremented for this path, so the task
sits in `ready/` forever, re-evaluated every poll.

On 2026-04-25 this silently stranded 5 growth-lead tasks for 21 minutes
before the dispatching agent gave up.

Fix: reject empty-routing at create time in `aofDispatch` (handler, not
Zod schema — per RESEARCH.md Pattern 5, the handler is the only place
`loadProjectManifest` defaulting can run). Optional default from
`project.owner.lead` (→ routing.agent) or `project.owner.team` (→
routing.team) before rejecting, EXCEPT when the owner value is the
`"system"` sentinel (case-insensitive) — that's the placeholder used
when no real owner is set (CONTEXT.md addendum Q3).

Purpose: prevent the silent-dispatch failure mode at its source. If the
task has no routing target, the caller gets a named error immediately,
not a task sitting in `ready/` for 21 minutes.

Output: routing-target validation in `aofDispatch`, project-owner
defaulting with `system`-sentinel handling, and a regression test suite
covering all 5 branches.
</objective>

    let agent = input.agent;
    let team = input.team;
    let role = input.role;

    if (!agent && !team && !role) {
      const projectId = input.project ?? ctx.projectId;
      if (projectId) {
        try {
          const manifest = await loadProjectManifest(ctx.store, projectId);
          const lead = manifest?.owner?.lead;
          const ownerTeam = manifest?.owner?.team;
          if (lead && lead.toLowerCase() !== "system") {
            agent = lead;
          } else if (ownerTeam && ownerTeam.toLowerCase() !== "system") {

          // loadProjectManifest returns null on read/parse errors and logs
