3. Replace every `console.log(...)` and `console.info(...)` with `log.info({ ...fields }, "message")`:

   - Example: `console.log(\`[AOF] Dispatching ${taskId}\`)` becomes `log.info({ taskId }, "dispatching task")`
4. Replace every `console.error(...)` with `log.error({ err, ...fields }, "message")` or `log.warn(...)` for caught/expected errors
5. Replace every `console.warn(...)` with `log.warn({ ...fields }, "message")`

For catch blocks:
- Silent catches (`catch (_err) {}` or `catch { }`) become `catch (err) { log.warn({ err, op: "operationName" }, "description (best-effort)"); }`
- Always use `err` as the field name (not `error` -- Pino's serializer is keyed to `err`)
- Include `op` field naming the operation, and any available identifiers (taskId, etc.)
- Keep the existing control flow (don't re-throw if the catch was intentionally swallowing)

Do NOT change any business logic, control flow, or function signatures. Only logging output changes.
