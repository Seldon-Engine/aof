**In `src/dispatch/executor.ts`:**

1. Rename `ExecutorResult` to `SpawnResult` (keep `ExecutorResult` as deprecated type alias for backward compat during transition).

2. Add `SessionStatus` interface:
```typescript
export interface SessionStatus {
  sessionId: string;
  alive: boolean;
  lastHeartbeatAt?: string;
  completedAt?: string;
}
```

3. Define `GatewayAdapter` interface replacing `DispatchExecutor`:
```typescript
export interface GatewayAdapter {
  /** Spawn an agent session for a task. */
  spawnSession(
    context: TaskContext,
    opts?: { timeoutMs?: number; correlationId?: string }
  ): Promise<SpawnResult>;

  /** Poll session status (heartbeat liveness check). */
  getSessionStatus(sessionId: string): Promise<SessionStatus>;

  /** Force-complete a stuck session. */
  forceCompleteSession(sessionId: string): Promise<void>;
}
```

4. Keep `DispatchExecutor` as a deprecated type alias: `export type DispatchExecutor = GatewayAdapter;` (temporary — removed in Task 2 after consumers migrate).

5. Refactor `MockExecutor` to `MockAdapter`:
   - Implement all three GatewayAdapter methods
   - `spawnSession()` replaces `spawn()` — same logic, plus tracks sessions in a `Map<string, MockSession>` for getSessionStatus
   - `getSessionStatus()` returns alive/heartbeat info from sessions map; supports `setSessionStale(sessionId)` for testing heartbeat timeout
   - `forceCompleteSession()` marks session as completed in the map
   - Keep `setShouldFail()`, `setShouldThrow()`, `clear()` test helpers
   - Add `completionDelayMs` constructor option for delayed auto-completion (default: 0 = instant)
   - Add `setAutoComplete(enabled: boolean)` to toggle auto-completion (default: true)
   - Keep `MockExecutor` as deprecated alias: `export const MockExecutor = MockAdapter;`

**In `src/openclaw/openclaw-executor.ts`:**

6. Rename class from `OpenClawExecutor` to `OpenClawAdapter`.

7. Change `implements DispatchExecutor` to `implements GatewayAdapter`.

8. Rename `spawn()` to `spawnSession()` — same internal logic (extensionAPI.runEmbeddedPiAgent fire-and-forget). Accept the new `opts` shape (with optional `correlationId`).

9. Add `sessionToTask` map: `private sessionToTask = new Map<string, string>()`. On successful spawn, store `sessionId -> taskId` mapping.

10. Implement `getSessionStatus(sessionId)`:
    - Look up `taskId` from `sessionToTask` map
    - If not found, return `{ sessionId, alive: false }`
    - Call `readHeartbeat(store, taskId)` — import from `../recovery/run-artifacts.js`
    - The adapter needs access to the `ITaskStore`. Add `store` as a constructor parameter (or receive it via a `setStore()` method if cleaner for the existing wiring in adapter.ts).
    - If no heartbeat, return `{ sessionId, alive: false }`
    - If `heartbeat.expiresAt` is in the past, return `{ sessionId, alive: false, lastHeartbeatAt: heartbeat.lastHeartbeat }`
    - Otherwise return `{ sessionId, alive: true, lastHeartbeatAt: heartbeat.lastHeartbeat }`

11. Implement `forceCompleteSession(sessionId)`:
    - Look up `taskId` from `sessionToTask` map
    - Call `markRunArtifactExpired(store, taskId, "force_completed")` — import from `../recovery/run-artifacts.js`
    - Remove from `sessionToTask` map
    - Log: `console.info(\`[AOF] Force-completed session ${sessionId} (task ${taskId})\`)`

12. Keep `OpenClawExecutor` as deprecated alias: `export const OpenClawExecutor = OpenClawAdapter;`

**IMPORTANT:** The existing `spawn()` logic in OpenClawExecutor is complex and works correctly — do NOT rewrite it. Just rename to `spawnSession()` and add the `correlationId` to opts signature. The only new logic is the three lines storing the sessionId -> taskId mapping after a successful spawn.
