<feature>
  <name>Gate-to-DAG lazy migration with TDD</name>
  <files>src/migration/gate-to-dag.ts, src/migration/__tests__/gate-to-dag.test.ts, src/store/task-store.ts</files>
  <behavior>
    - Test 1: Task with no gate field and no workflow field returns unchanged (no-op)
    - Test 2: Task already having workflow field returns unchanged (skip)
    - Test 3: Task with gate field (not in-flight, no current) converts gates to linear DAG hops with correct dependsOn chain
    - Test 4: Gate fields (gate, gateHistory, reviewContext) are cleared after conversion (mutual exclusivity)
    - Test 5: In-flight task at gate N maps gates 0..N-1 to complete, gate N to dispatched (when task status is in-progress), gates N+1..end to pending
    - Test 6: In-flight task at gate N with task status not in-progress maps gate N to ready
    - Test 7: Gate with canReject=true maps to hop with canReject=true and rejectionStrategy="origin"
    - Test 8: Gate with timeout and escalateTo maps to hop with matching timeout and escalateTo
    - Test 9: Gate with simple `when` expression (tags.includes('X')) converts to JSON DSL condition ({op: "has_tag", value: "X"})
    - Test 10: Gate with complex/unparseable `when` expression logs warning and skips condition (hop always activates)
    - Test 11: Converted DAG passes validateDAG (no cycles, all hops reachable)
    - Test 12: Migration is idempotent (running twice on same task produces same result)
  </behavior>
  <implementation>
    1. Create src/migration/gate-to-dag.ts:
       - Export `migrateGateToDAG(task, workflowConfig?)` function
       - Detect gate format: check task.frontmatter.gate exists and task.frontmatter.workflow does not
       - Convert gate array to hop array: each gate becomes a hop with dependsOn=[previous hop id] (linear chain)
       - Map gate fields: id, role, canReject, timeout, escalateTo, description
       - Convert gate `when` string expressions to JSON DSL conditions:
         - `tags.includes('X')` -> `{op: "has_tag", value: "X"}`
         - `!tags.includes('X')` -> `{op: "not", condition: {op: "has_tag", value: "X"}}`
         - `||` combinations -> `{op: "or", conditions: [...]}`
         - `&&` combinations -> `{op: "and", conditions: [...]}`
         - `metadata.X > N` -> `{op: "gt", field: "metadata.X", value: N}`
         - Unparseable expressions: log warning, skip condition
       - Map in-flight position: use gate.current to determine hop statuses (complete/dispatched/ready/pending)
       - Call initializeWorkflowState() for state, then overlay position mapping
       - Clear gate fields: set gate=undefined, gateHistory=[], reviewContext=undefined
       - Return modified task (mutation, caller handles persistence)

    2. Hook into src/store/task-store.ts:
       - In get() method: after parseTaskFile(), check if task has gate fields. If so, call migrateGateToDAG(), then write back to disk atomically using existing serializeTask + writeFileAtomic pattern
       - In list() method: same migration check for each loaded task
       - The migration is one-time per task: after write-back, next load sees DAG format

    IMPORTANT per user decision:
    - Gate evaluation code kept with deprecation markers (NOT removed) -- safety net for edge cases
    - Dual-mode evaluator remains as fallback during migration period
    - One-time conversion per task -- next load sees DAG format natively
  </implementation>
</feature>
