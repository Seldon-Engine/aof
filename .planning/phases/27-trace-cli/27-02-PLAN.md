<task type="auto" tdd="true">
  <name>Task 1: Trace CLI command with DAG hop correlation</name>
  <files>src/cli/commands/trace.ts, src/cli/commands/__tests__/trace.test.ts</files>
  <behavior>
    - registerTraceCommand(program) registers "trace &lt;task-id&gt;" command with --debug, --json, --project options
    - When task not found by prefix, prints "Task not found: {id}" to stderr and sets exitCode=1
    - When no trace files found, prints "No traces found for task {id}" to stderr (or "No traces found for task {id}. Traces are captured after agent sessions complete." for better UX)
    - When --json flag set, errors go to stderr and only valid JSON goes to stdout
    - For non-workflow tasks, calls formatTraceSummary or formatTraceDebug with no hopMap
    - For workflow tasks (task.frontmatter.workflow exists), builds HopInfo[] by mapping workflow.definition.hops to traces via correlationId or sequential attempt ordering, then passes hopMap to formatter
    - Default --project is "_inbox"
  </behavior>
  <action>
    Create `src/cli/commands/trace.ts` exporting `registerTraceCommand(program: Command): void`.

    Command structure:
    ```
    program
      .command("trace <task-id>")
      .description("Show trace of agent activity for a task")
      .option("--debug", "Show full tool call details and reasoning text")
      .option("--json", "Output structured trace data as JSON")
      .option("--project <id>", "Project ID", "_inbox")
      .action(async (taskId, opts) => { ... })
    ```

    Action implementation:
    1. Dynamic import createProjectStore from "../project-utils.js"
    2. Get root from `program.opts()["root"]`
    3. Create store via createProjectStore({ projectId: opts.project, vaultRoot: root })
    4. Look up task via `store.getByPrefix(taskId)` -- error if not found
    5. Resolve task directory: `join(projectRoot, "state", "runs", task.frontmatter.id)`
    6. Call `readTraceFiles(taskDir)` -- error if empty
    7. Build hopMap if task has workflow:
       - Read `task.frontmatter.workflow.definition.hops` for hop IDs and roles
       - Read `task.frontmatter.workflow.state.hops` for correlationIds
       - For each definition hop, find matching traces by correlationId (preferred) or by sequential attempt ordering (fallback)
       - Build `HopInfo[]` with traceIndices pointing into the traces array
    8. Dispatch to formatter based on flags:
       - `--json`: `console.log(formatTraceJson(traces))` and return
       - `--debug`: `console.log(formatTraceDebug(fullTaskId, traces, hopMap))`
       - default: `console.log(formatTraceSummary(fullTaskId, traces, hopMap))`

    For the hop correlation logic, create a helper `buildHopMap(workflow, traces)` inside the module:
    - Iterate workflow.definition.hops
    - For each hop, check if workflow.state.hops[hopId].correlationId matches any trace's sessionId
    - If no correlationId match, fall back to sequential ordering (first trace = first hop, etc.)
    - Unmatched traces go into an "unassigned" section

    Test file tests the buildHopMap logic with fixture data and tests error handling paths. Use vi.mock for store and file system operations.
  </action>
