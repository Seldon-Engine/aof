<feature>
  <name>Feature 1: Trace Reader</name>
  <files>src/trace/trace-reader.ts, src/trace/__tests__/trace-reader.test.ts</files>
  <behavior>
    - readTraceFiles(taskDir) returns TraceSchema[] sorted by attemptNumber when trace-N.json files exist
    - readTraceFiles(taskDir) returns empty array when directory does not exist
    - readTraceFiles(taskDir) returns empty array when directory has no trace-*.json files
    - readTraceFiles(taskDir) skips corrupted/malformed trace files silently (does not throw)
    - readTraceFiles(taskDir) filters to only trace-N.json pattern (ignores run.json, run_result.json, etc.)
    - Sorting is numeric: trace-1, trace-2, ..., trace-10 (not lexicographic)
  </behavior>
  <implementation>
    Create `src/trace/trace-reader.ts` exporting `readTraceFiles(taskDir: string): Promise<TraceSchema[]>`.

    Implementation:
    1. Try `readdir(taskDir)` -- catch and return [] if dir missing
    2. Filter entries matching `/^trace-\d+\.json$/`
    3. Sort by extracted number (parseInt on the digit group)
    4. For each file: read, JSON.parse, TraceSchema.parse -- wrap in try/catch, skip failures
    5. Return array of parsed traces

    Import TraceSchema from `../schemas/trace.js`. Use `node:fs/promises` readdir and readFile, `node:path` join.

    Test file creates fixture trace data as in-memory objects (use JSON.stringify to write temp files in beforeEach), not the JSONL fixtures from Phase 26 (those are for session-parser, not trace-reader).
  </implementation>
</feature>

<feature>
  <name>Feature 2: Trace Formatter</name>
  <files>src/trace/trace-formatter.ts, src/trace/__tests__/trace-formatter.test.ts</files>
  <behavior>
    Summary mode (formatTraceSummary):
    - Shows task ID, then per-attempt sections
    - Each attempt shows: model (provider), duration (human-readable), tool call count, no-op flag, capture mode
    - Each attempt shows tool usage breakdown: tool names with counts (e.g., "Read x5, Write x3")
    - Duration formatted as "Xs" for <60s, "Xm Ys" for >=60s
    - When traces array is empty, returns "No traces found." message

    Debug mode (formatTraceDebug):
    - Includes everything from summary
    - Additionally shows each tool call with full input and output (when available)
    - Shows reasoning text array if present
    - When trace was captured in summary mode (meta.mode === "summary"), appends a note: "Trace captured in summary mode. Re-run task with debug=true for full details."

    JSON mode (formatTraceJson):
    - Returns JSON.stringify of trace array (single trace = object, multiple = array)
    - No decorative text, only valid JSON
    - Pretty-printed with 2-space indent

    DAG hop grouping (formatTraceSummary and formatTraceDebug with hopMap):
    - Optional hopMap parameter: array of { hopId, role, traceIndices } mapping traces to hops
    - When hopMap provided, groups traces under hop headers: "Hop: {hopId} (role: {role})"
    - When hopMap not provided, displays flat list of attempts

    formatDuration helper:
    - 0ms -> "0s"
    - 5000ms -> "5s"
    - 65000ms -> "1m 5s"
    - 3661000ms -> "61m 1s"
  </behavior>
  <implementation>
    Create `src/trace/trace-formatter.ts` exporting:
    - `formatTraceSummary(taskId: string, traces: TraceSchema[], hopMap?: HopInfo[]): string`
    - `formatTraceDebug(taskId: string, traces: TraceSchema[], hopMap?: HopInfo[]): string`
    - `formatTraceJson(traces: TraceSchema[]): string`
    - `formatDuration(ms: number): string`
    - `interface HopInfo { hopId: string; role: string; traceIndices: number[] }`

    All formatters are pure functions: take typed data, return strings. No I/O, no side effects.

    Summary format per attempt:
    ```
    Attempt N (trace-N.json)
      Model: {model} ({provider})
      Duration: {formatted}
      Tool Calls: {count}
      No-op: {yes/no}
      Mode: {summary/debug}

      Tools Used:
        {name}    x{count}
    ```

    Debug format adds after each attempt's summary:
    ```
      Tool Call Details:
        1. {name}
           Input: {input}
           Output: {output or "(not captured)"}

      Reasoning:
        {reasoning text blocks}
    ```

    For JSON mode: `traces.length === 1 ? traces[0] : traces` then `JSON.stringify(output, null, 2)`.

    Tests use fixture trace objects built inline (not from disk). Test each mode independently. Test hopMap grouping with a 2-hop scenario.
  </implementation>
</feature>
