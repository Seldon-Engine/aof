  <behavior>
    - handleDAGHopCompletion with done run result: finds dispatched hop, maps to HopEvent {outcome: "complete"}, calls evaluateDAG, persists new state atomically, returns readyHops
    - handleDAGHopCompletion with blocked/error run result: maps to HopEvent {outcome: "failed"}, evaluateDAG cascades skips, persists atomically
    - handleDAGHopCompletion with run result notes: notes become hop result field ({notes: "..."})
    - handleDAGHopCompletion when no dispatched hop found: logs warning, returns empty readyHops (defensive edge case)
    - handleDAGHopCompletion when DAG completes (evaluateDAG returns taskStatus): returns dagComplete=true
    - handleDAGHopCompletion when autoAdvance=false on completed hop: returns reviewRequired=true
    - dispatchDAGHop: builds HopContext, builds TaskContext with hopContext, calls spawnSession, on success sets hop to dispatched with startedAt/agent/correlationId and persists atomically
    - dispatchDAGHop: on spawnSession failure, hop stays "ready" (not dispatched), returns false
    - dispatchDAGHop: sets hop status to dispatched ONLY after spawnSession succeeds (per user decision)
  </behavior>
  <action>
    **Create `src/dispatch/dag-transition-handler.ts`** mirroring `gate-transition-handler.ts` structure:

    **Import dependencies:**
    - `evaluateDAG`, `DAGEvaluationInput`, `HopEvent`, `EvalContext` from `./dag-evaluator.js`
    - `buildHopContext`, `HopContext` from `./dag-context-builder.js`
    - `buildConditionContext` from `./dag-condition-evaluator.js`
    - `serializeTask` from `../store/task-store.js`
    - `writeFileAtomic` from `write-file-atomic`
    - `Task` from `../schemas/task.js`
    - `TaskContext`, `GatewayAdapter`, `SpawnResult` from `./executor.js`
    - `EventLogger` from `../events/logger.js`
    - `ITaskStore` from `../store/interfaces.js`
    - `RunResult` from `../schemas/run-result.js`

    **Helper: `findDispatchedHop(task: Task): string | undefined`**
    Find the hop with status `"dispatched"` in `task.frontmatter.workflow.state.hops`. Return undefined if none found.

    **Helper: `mapRunResultToHopEvent(runResult: RunResult, hopId: string): HopEvent`**
    - `runResult.outcome === "done"` → `{ hopId, outcome: "complete", result: runResult.notes ? { notes: runResult.notes } : undefined }`
    - Any other outcome → `{ hopId, outcome: "failed", result: runResult.notes ? { notes: runResult.notes } : undefined }`
    - Per user decision: run result notes/data become the hop's `result` field.

    **Helper: `buildEvalContext(task: Task): EvalContext`**
    Collect hop results from all completed hops in `workflow.state.hops` (if hopState.result exists, include it). Build task metadata from frontmatter (status, tags, priority, routing). Return `EvalContext` compatible with `evaluateDAG`.

    **Helper: `persistWorkflowState(task: Task, newState: WorkflowState): Promise<void>`**
    Update `task.frontmatter.workflow.state` with `newState`, set `task.frontmatter.updatedAt` to ISO timestamp, call `writeFileAtomic(task.path!, serializeTask(task))`.

    **Export: `handleDAGHopCompletion(store, logger, task, runResult): Promise<{ readyHops: string[]; dagComplete: boolean; reviewRequired: boolean }>`**
    1. Find dispatched hop via `findDispatchedHop(task)`. If none found, log warning, return `{ readyHops: [], dagComplete: false, reviewRequired: false }`.
    2. Map run result to HopEvent via `mapRunResultToHopEvent`.
    3. Build EvalContext via `buildEvalContext`.
    4. Call `evaluateDAG({ definition: task.frontmatter.workflow!.definition, state: task.frontmatter.workflow!.state, event: hopEvent, context: evalContext })`.
    5. Persist new state via `persistWorkflowState(task, evalResult.state)`.
    6. Log hop transition event via `logger.log("dag.hop_completed", ...)`.
    7. If `evalResult.taskStatus` is set (DAG complete/failed), return `dagComplete: true`.
    8. Check if the completed hop has `autoAdvance: false` in the definition AND the outcome was "complete" → return `reviewRequired: true`.
    9. Return `{ readyHops: evalResult.readyHops, dagComplete: evalResult.taskStatus !== undefined, reviewRequired }`.

    **Export: `dispatchDAGHop(store, logger, config, executor, task, hopId): Promise<boolean>`**
    1. Find hop definition from `task.frontmatter.workflow!.definition.hops`.
    2. Build `HopContext` via `buildHopContext(task, hopId)`.
    3. Build `TaskContext` with hop context injected:
       ```typescript
       const context: TaskContext = {
         taskId: task.frontmatter.id,
         taskPath: task.path!,
         agent: hop.role, // Role → agent resolution
         priority: task.frontmatter.priority,
         routing: { role: hop.role },
         projectId: task.frontmatter.project,
         hopContext,
       };
       ```
    4. Call `executor.spawnSession(context, { timeoutMs: config.spawnTimeoutMs ?? 30_000, correlationId: randomUUID() })`.
    5. **On success (per user decision: set dispatched ONLY after spawn succeeds):**
       - Update hop state: `{ status: "dispatched", startedAt: new Date().toISOString(), agent: hop.role, correlationId: result.sessionId }`.
       - Persist state atomically via `persistWorkflowState`.
       - Log dispatch event.
       - Return `true`.
    6. **On failure (per user decision: hop stays "ready" for retry):**
       - Log dispatch failure.
       - Return `false`.

    **Tests:** Use vitest with mocked store, logger, and executor. Test all behavior cases listed above. Mock `evaluateDAG` and `writeFileAtomic` for unit isolation. Create test helper to build minimal Task objects with workflow frontmatter.
