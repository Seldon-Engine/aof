In `src/store/task-store.ts`, the `get()` method (line 225-227) and `list()` method (line 298-300) both call `migrateGateToDAG(task)` without the `workflowConfig` second argument. The migration function guards on missing config and returns the task unchanged, making migration a no-op.

**Fix for `get()` (around line 225):**

The task-store has `this.projectRoot` which is the project directory. To load the workflow config, read the project manifest (project.yaml) and extract `workflow.gates` from it. Use a helper method on the class to avoid duplication.

Add a private method to FilesystemTaskStore:
```typescript
/**
 * Load workflow config from project manifest for gate-to-DAG migration.
 * Returns undefined if manifest doesn't exist or has no workflow gates.
 */
private async loadWorkflowConfig(): Promise<import("../migration/gate-to-dag.js").WorkflowConfig | undefined> {
  try {
    const { readFile } = await import("node:fs/promises");
    const { parse } = await import("yaml");
    const manifestPath = join(this.projectRoot, "project.yaml");
    const raw = await readFile(manifestPath, "utf-8");
    const manifest = parse(raw);
    if (manifest?.workflow?.gates?.length) {
      return {
        name: manifest.workflow?.name ?? "default",
        gates: manifest.workflow.gates,
      };
    }
  } catch {
    // No manifest or parse error — migration not possible
  }
  return undefined;
}
```

Then update both call sites:

In `get()` (line 225-227), change:
```typescript
if ((task.frontmatter as any).gate && !task.frontmatter.workflow) {
  migrateGateToDAG(task);
  await writeFileAtomic(filePath, serializeTask(task));
}
```
To:
```typescript
if ((task.frontmatter as any).gate && !task.frontmatter.workflow) {
  const workflowConfig = await this.loadWorkflowConfig();
  migrateGateToDAG(task, workflowConfig);
  if (task.frontmatter.workflow) {
    await writeFileAtomic(filePath, serializeTask(task));
  }
}
```

Note: only write back if migration actually succeeded (task now has workflow field). If workflowConfig is missing, migrateGateToDAG returns unchanged and we skip the write.

In `list()` (line 298-300), apply the same pattern. Load workflowConfig ONCE before the loop (not per-task):
```typescript

```

Then inside the migration block:
```typescript
if ((task.frontmatter as any).gate && !task.frontmatter.workflow) {
  if (!workflowConfigLoaded) {
    workflowConfig = await this.loadWorkflowConfig();
    workflowConfigLoaded = true;
  }
  migrateGateToDAG(task, workflowConfig);
  if (task.frontmatter.workflow) {
    await writeFileAtomic(filePath, serializeTask(task));
  }
}
```

The lazy-load pattern ensures: (a) config is only loaded if a gate task is actually found, (b) config is loaded at most once per list() call.
