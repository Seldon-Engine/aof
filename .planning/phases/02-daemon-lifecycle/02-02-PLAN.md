  <name>Task 1: Create service file generation module</name>
  <files>src/daemon/service-file.ts, src/daemon/__tests__/service-file.test.ts</files>
  <action>
Create `src/daemon/service-file.ts` with the following exports:

**`generateLaunchdPlist(config: ServiceFileConfig): string`**
Generate a valid launchd plist XML string. Template must include:
- `Label`: `"ai.openclaw.aof"` (constant)
- `ProgramArguments`: `["node", "{path-to-aof-daemon-binary}", "--root", "{dataDir}"]`. Resolve the aof-daemon binary path via `require.resolve("aof/dist/daemon/index.js")` or by computing relative to the package. The `--root` arg points to the AOF data directory.
- `RunAtLoad`: true (start on install)
- `KeepAlive`: true (restart on crash)
- `ThrottleInterval`: 5 (per user decision: 5-second restart delay)
- `StandardOutPath`: `join(dataDir, "logs", "daemon-stdout.log")`
- `StandardErrorPath`: `join(dataDir, "logs", "daemon-stderr.log")`
- `EnvironmentVariables`: `{ AOF_ROOT: dataDir }` ‚Äî the daemon reads this
- `WorkingDirectory`: dataDir

**`generateSystemdUnit(config: ServiceFileConfig): string`**
Generate a valid systemd unit file string. Template must include:
```ini
[Unit]
Description=AOF Daemon - Agentic Ops Fabric Scheduler
After=network.target

[Service]
Type=simple
ExecStart=/usr/bin/env node {path-to-aof-daemon-binary} --root {dataDir}
Restart=on-failure
RestartSec=5
Environment=AOF_ROOT={dataDir}
WorkingDirectory={dataDir}
StandardOutput=append:{dataDir}/logs/daemon-stdout.log
StandardError=append:{dataDir}/logs/daemon-stderr.log

[Install]
WantedBy=default.target
```

**`ServiceFileConfig` interface:**
```typescript
export interface ServiceFileConfig {
  dataDir: string;           // AOF data directory
  nodeBinary?: string;       // Path to node binary (default: process.execPath)
  daemonBinary?: string;     // Path to aof-daemon entry (auto-resolved)
  extraArgs?: string[];      // Additional CLI args
  extraEnv?: Record<string, string>;  // Additional env vars
}
```

**`getServiceFilePath(platform: NodeJS.Platform): string`**
Returns the path where the service file should be written:
- `darwin`: `join(homedir(), "Library", "LaunchAgents", "ai.openclaw.aof.plist")`
- `linux`: `join(homedir(), ".config", "systemd", "user", "ai.openclaw.aof.service")`
- Other: throw with clear message

**`installService(config: ServiceFileConfig): Promise<InstallResult>`**
1. Detect platform (`process.platform`)
2. Generate the appropriate service file content
3. Create parent directory if needed (`mkdir -p`)
4. Write the service file
5. On macOS: run `launchctl load {plistPath}` via `execSync`
6. On Linux: run `systemctl --user daemon-reload && systemctl --user enable --now ai.openclaw.aof` via `execSync`
7. Return `{ platform, servicePath, started: true }`

**`uninstallService(): Promise<void>`**
1. On macOS: `launchctl unload {plistPath}` then delete plist file
2. On Linux: `systemctl --user disable --now ai.openclaw.aof` then delete unit file
3. Clean up socket file and PID file if they exist

<task type="auto">
  <name>Task 2: Wire install/uninstall commands into CLI and add config validation</name>
  <files>src/cli/commands/daemon.ts</files>
  <action>
Add `install` and `uninstall` subcommands to the existing daemon command group in `src/cli/commands/daemon.ts`.

**`aof daemon install` command:**
1. Accept options: `--data-dir <path>` (default: AOF_ROOT from program opts)
2. **Validate config first** (per user decision): Check that the data directory exists, has a valid task store structure (at minimum: `tasks/` directory exists or can be created), and the org chart is loadable if present. If validation fails, print clear error and exit code 1.
3. Ensure logs directory exists: `mkdir -p {dataDir}/logs`
4. Call `installService({ dataDir })` from `src/daemon/service-file.ts`
5. Wait 2 seconds, then check if daemon is running:
   - Try to self-check the health socket using `selfCheck(join(dataDir, "daemon.sock"))` from `src/daemon/server.ts`
   - If healthy: print success with PID and socket path
   - If not: print warning "Service file installed but daemon may not have started yet. Check `aof daemon status`."
6. Output format:
   ```
   Installing AOF daemon...

     Platform: macOS (launchd)
     Service file: ~/Library/LaunchAgents/ai.openclaw.aof.plist
     Data directory: /path/to/aof
     Socket: /path/to/aof/daemon.sock

   Daemon installed and started.
   ```

**`aof daemon uninstall` command:**
1. Call `uninstallService()` from `src/daemon/service-file.ts`
2. Clean up: remove socket file, PID file if they exist
3. Output: "Daemon uninstalled. Service file removed."
4. Exit code 0 on success, 1 on error

**Remove the old `start` and `restart` commands** ‚Äî they are replaced by `install`/`uninstall` which manage the daemon via the OS supervisor. Keep `stop` (sends SIGTERM via the OS supervisor or direct PID kill) and `status` (will be redesigned in Plan 03).
