  <files>src/cli/commands/smoke.ts, src/cli/commands/__tests__/smoke.test.ts</files>

    - Test 2: Version check fails when package.json is missing
    - Test 3: Schema check fails when project.yaml has invalid schema (missing required fields)
    - Test 4: Task store check fails when tasks directory is missing
    - Test 5: Org chart check fails when org-chart.yaml is malformed
    - Test 6: Migration check reports correct applied migration count
    - Test 7: Workflow template check fails when defaultWorkflow references nonexistent template
    - Test 8: Overall result is pass only when all individual checks pass
  </behavior>
  <action>
Create `src/cli/commands/smoke.ts` with:

1. Define a `SmokeCheck` interface: `{ name: string; run: (root: string) => Promise<SmokeResult> }` and `SmokeResult`: `{ pass: boolean; detail: string }`.

2. Implement individual check functions (each returns `SmokeResult`):
   - **versionCheck**: Read `package.json` from root, extract `version` field. Pass if version exists and is a non-empty string. Detail shows the version string.
   - **schemaCheck**: Glob for `Projects/*/project.yaml` files, parse each with `ProjectManifest.safeParse()`. Pass if all valid (or no projects found). Detail shows count validated or specific errors.
   - **taskStoreCheck**: Check that `Projects/*/tasks/` directory exists for at least one project. Try reading files from task status directories (backlog, ready, in-progress, done, blocked, review). Pass if directory structure is accessible. Detail shows task count or error.
   - **orgChartCheck**: Look for `org/org-chart.yaml` in root. Parse with yaml + `OrgChart.safeParse()`. Pass if valid or no org chart exists. Detail shows unit count or error.
   - **migrationCheck**: Call `getMigrationHistory(root)` from `src/packaging/migrations.ts`. Pass if history is readable (even if empty -- fresh installs have no migrations). Detail shows count of applied migrations.
   - **workflowCheck**: For each project.yaml, check that if `defaultWorkflow` is set, the referenced template exists in `workflowTemplates`. Pass if all references resolve. Detail shows template count or broken references.

3. Export `runSmokeChecks(root: string): Promise<{ checks: Array<{name: string} & SmokeResult>; allPassed: boolean }>` that runs all checks in sequence and aggregates results.

4. Export `registerSmokeCommand(program: Command): void` that registers `aof smoke` as a Commander subcommand:
   - Uses `program.opts()["root"]` for root directory
   - Runs `runSmokeChecks(root)`
   - Prints checklist format: green checkmark + name + detail for pass, red X + name + detail for fail
   - Use ANSI escape codes matching setup.ts pattern: `\x1b[32m` for green pass, `\x1b[31m` for red fail, `\x1b[0m` reset
   - Sets `process.exitCode = 1` if any check fails (do NOT call process.exit -- let Commander finish cleanly)

5. Do NOT import `readPackageVersion` from setup.ts (it's not exported). Inline the trivial version read: `JSON.parse(await readFile(join(root, "package.json"), "utf-8")).version`.

6. Import `getMigrationHistory` from `../../packaging/migrations.js`. Import `ProjectManifest` from `../../schemas/project.js`. Import `OrgChart` from `../../schemas/org-chart.js`. Import `parse as parseYaml` from `yaml`. Import `readFile, readdir, access` from `node:fs/promises`.

Create `src/cli/commands/__tests__/smoke.test.ts` following the temp-directory pattern from existing tests:
- Use `mkdtemp` for isolated test directories
- Scaffold valid data dirs with: `package.json`, `Projects/test-project/project.yaml`, `org/org-chart.yaml`, `Projects/test-project/tasks/{backlog,ready,done}/` directories, `.aof/migrations.json`
- Test each failure mode by omitting or corrupting the relevant file
- Test the aggregated `runSmokeChecks()` function, not the CLI output formatting
