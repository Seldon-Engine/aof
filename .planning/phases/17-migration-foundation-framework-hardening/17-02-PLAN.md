  <behavior>
    Migration 001 (defaultWorkflow):
    - Discovers all project.yaml files under `{aofRoot}/Projects/*/project.yaml`
    - For each: reads with `parseDocument()`, checks if `defaultWorkflow` already exists (skip if so -- idempotent)
    - If `workflowTemplates` exists and has entries, picks the first key name and sets `defaultWorkflow` to it
    - If no `workflowTemplates`, skips the project (stays bare-task)
    - Writes back with `writeFileAtomic`, preserving YAML comments
    - Logs "001-default-workflow-template applied" with checkmark

    Migration 002 (gate-to-DAG batch):
    - Discovers all projects under `{aofRoot}/Projects/*/`
    - For each project, walks all 8 status directories (backlog, ready, in-progress, blocked, review, done, cancelled, deadletter)
    - For each .md task file: parses, checks for `gate` field without `workflow` (skip if already migrated -- idempotent)
    - Calls existing `migrateGateToDAG(task, workflowConfig)` for conversion
    - Writes back with `writeFileAtomic`
    - Logs total converted count

    Migration 003 (version metadata):
    - Reads existing `.aof/channel.json` if present (for previousVersion)
    - Fresh install (no existing): writes `{ version, channel: "stable", installedAt }`
    - Upgrade (existing): writes `{ version, previousVersion: existing.version, channel: "stable", upgradedAt }`
    - Idempotent: if existing channel.json has same version, skip
    - Writes with `writeFileAtomic`

    Tests:
    - 001: project with templates gets defaultWorkflow set; project without templates is skipped; already-set project is skipped; YAML comments preserved
    - 002: task with gate field gets converted; task without gate skipped; already-migrated task skipped
    - 003: fresh install writes installedAt; upgrade writes previousVersion and upgradedAt; same-version skip
