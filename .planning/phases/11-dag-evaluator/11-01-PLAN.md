<feature>
  <name>DAG Condition Evaluator</name>
  <files>src/dispatch/dag-condition-evaluator.ts, src/dispatch/__tests__/dag-condition-evaluator.test.ts</files>
  <behavior>
    --- Condition Context Building ---
    - buildConditionContext(state, taskMeta) returns a flat context object with hop results under "hops.{hopId}.result.{field}" and task metadata under "task.{field}"

    --- Dot-Path Field Resolution ---
    - getField({a: {b: {c: 42}}}, "a.b.c") -> 42
    - getField({a: {b: 1}}, "a.b.c") -> undefined (missing nested field)
    - getField({}, "x.y.z") -> undefined (empty object)
    - getField(null, "a") -> undefined (null root)

    --- Comparison Operators ---
    - eq: getField(ctx, field) === value; undefined field -> false (unless value is also undefined)
    - neq: getField(ctx, field) !== value; undefined field -> true (undefined !== anything-else)
    - gt: (getField(ctx, field) as number) > value; undefined -> false
    - gte: (getField(ctx, field) as number) >= value; undefined -> false
    - lt: (getField(ctx, field) as number) < value; undefined -> false
    - lte: (getField(ctx, field) as number) <= value; undefined -> false

    --- Collection Operators ---
    - in: expr.value.includes(getField(ctx, field)); undefined field -> false
    - has_tag: special operator, checks ctx.task.tags.includes(expr.value); missing tags -> false

    --- DAG-Aware Operators ---
    - hop_status: special operator, reads from ctx.hopStates[expr.hop]?.status === expr.status; hop_status reads from live WorkflowState.hops, NOT from condition context field lookup

    --- Logical Operators ---
    - and: conditions.every(c => evaluateCondition(c, ctx))
    - or: conditions.some(c => evaluateCondition(c, ctx))
    - not: !evaluateCondition(condition, ctx)
    - and with empty array -> true (vacuous truth)
    - or with empty array -> false

    --- Literal Operators ---
    - true -> true
    - false -> false

    --- Unknown Operators ---
    - Unknown op -> false (defensive, shouldn't happen with Zod validation)
  </behavior>
  <implementation>
    Implement as a single module with three exported functions:

    1. `getField(obj, path)` -- dot-path field resolution (5-line split('.') walker per RESEARCH.md)
    2. `buildConditionContext(state, taskMeta)` -- builds flat context for dot-path resolution. Merges hop results under "hops.{hopId}" prefix, task metadata under "task." prefix
    3. `evaluateCondition(expr, ctx)` -- per-operator dispatch table (`Record<string, handler>`) mapping each `ConditionExprType.op` to its evaluation function

    The `ConditionContext` interface wraps: `context` (flat dot-path-resolvable object), `hopStates` (live WorkflowState.hops map for hop_status operator), `task` (metadata with tags array for has_tag operator).

    Follow CONTEXT.md locked decisions:
    - Missing fields resolve to undefined, treated as falsy
    - hop_status reads directly from WorkflowState.hops map (special operator)
    - has_tag is a special operator checking ctx.task.tags directly
    - Per-operator dispatch table (not switch/if-chain)
  </implementation>
</feature>
