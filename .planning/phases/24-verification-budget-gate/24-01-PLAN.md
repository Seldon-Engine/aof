  <action>
Create `src/context/__tests__/context-budget-gate.test.ts` with a `describe("context budget gate")` block.

**Constants at top of file:**
- `BUDGET_CEILING_TOKENS` -- set to the current measured full-tier total (SKILL.md tokens + tool description tokens) plus 25% headroom, rounded to a clean number. This is the hard ceiling for regression detection.
- `PRE_V14_SKILL_BASELINE_TOKENS` -- measure from git history: run `git show d3d118b:skills/aof/SKILL.md` (the pre-v1.4 commit), pipe through `estimateTokens()`, and hardcode the result. Phase 22 summary measured 13645 chars / 3411 tokens. Use 3411 as the baseline. **Do NOT use 3266** -- that was an earlier incorrect estimate; Phase 22 verified 3411 from the actual file.

**How to measure current values at test time:**
1. Read `skills/aof/SKILL.md` from disk using `fs.readFile` (use `process.cwd()` to find project root -- vitest runs from repo root)
2. Compute `skillTokens = estimateTokens(skillContent)` -- this is the SKILL.md-only token count
3. Read `src/mcp/tools.ts` from disk, extract all `description: "..."` string values using a regex like `/description:\s*"([^"]+)"/g`
4. Compute `toolDescTokens = estimateTokens(allDescriptionsConcatenated)`
5. Compute `totalTokens = skillTokens + toolDescTokens` -- used for the budget ceiling test

**Tests:**
1. `it("full-tier context stays under budget ceiling")` -- assert `totalTokens <= BUDGET_CEILING_TOKENS`. This guards the combined SKILL.md + tool descriptions against regression.
2. `it("gate catches regressions")` -- compute what the total would be if SKILL.md were 4x its current size, assert it would exceed the ceiling. Sanity check that the ceiling is meaningful.
3. `it("achieves 50%+ reduction from pre-v1.4 baseline")` -- assert `skillTokens < PRE_V14_SKILL_BASELINE_TOKENS * 0.5`. **CRITICAL: Compare SKILL.md tokens only on BOTH sides.** Tool descriptions were already one-liners pre-v1.4 and are unchanged, so including them on the "after" side without a corresponding "before" amount would be an apples-to-oranges comparison. SKILL.md alone went from 3411 to ~1666 tokens (51.2% reduction), which comfortably passes the `< 0.5` threshold (1666 < 1705.5).

**Style:** Follow `src/context/__tests__/budget.test.ts` patterns -- `import { describe, it, expect } from "vitest"`, no external mocking. Use `import * as fs from "node:fs/promises"` and `import * as path from "node:path"` for file reads.

**Important:** Use `process.cwd()` to resolve file paths (vitest runs from project root). Do NOT use `__dirname` (not available in ESM).
